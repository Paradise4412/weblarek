# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является классом для всех компонентов интерфейса

Конструктор:  
`constructor(container: HTMLElement)` принимает ссылку на DOM элемент за отображение, которого он отвечает
Поля класса:  
`container: HTMLElement` поле для хранения корневого DOM элемента компонента
Методы класса:  
`render(data?: Partial<T>): HTMLElement` принимает данные которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. 
`setImage(element: HTMLImageElement, src: string, alt?: string): void` утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` в конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов
Поля класса:  
`baseUrl: string` базовый адрес сервера  
`options: RequestInit` объект с заголовками которые будут использованы для запросов
Методы:   
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` принимает объект с данными которые будут переданы в JSON в теле запроса и отправляет эти данные на ендпоинт переданный как параметр при вызове метода
`handleResponse(response: Response): Promise<object>` проверяет ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных

#### Класс EventEmitter

Конструктор класса не принимает параметров
Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)`  хранит коллекцию подписок на события 
ключи коллекции - названия событий, значения - коллекция функций обработчиков
Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` подписка на событие
`emit<T extends object>(event: string, data?: T): void` инициализация события 

## Данные

Интерфейсы данных описывают поля которые будут храниться в приложении и давать подсказки при работе 

interface IProduct {
	id: string
	description: string
	image: string
	title: string
	category: string
	price: number | null
}

interface IBuyer {
	payment: TPayment
	email: string
	phone: string
	address: string
}

`IProduct` описывает поля товара 
`id` хранится уникальный идентификатор товара
`description` хранится текст описания
`image` хранится путь к картинке товара
`title` хранится название товара 
`category` хранится категория товара
`price` хранится цена товара или `null` если цена не задана

`IBuyer` описывает данные покупателя
`payment` хранится выбранный способ оплаты
`email` хранится электронная почта покупателя 
`phone` хранится телефон 
`address` хранится адрес доставки

## Модели данных
Первый класс хранит все доступные товары и выбранный товар
Второй класс хранит товары в корзине
Третий класс хранит данные покупателя и валидацию этих данных

### ProductCatalog
Конструктор
`constructor(products: IProduct[] = [])`

Поля класса
`products: IProduct[]` массив всех доступных товаров
`selectedProduct: IProduct | null` товар для подробного отображения

Методы класса
`setProducts(products: IProduct[]): void` сохраняет массив товаров переданный в параметре
`getProducts(): IProduct[]` возвращает массив товаров из модели
`getProductById(id: string): IProduct | undefined` ищет и возвращает товар по его `id`

### Cart

Конструктор
`constructor(items: IProduct[] = [])`

Поля класса
`items: IProduct[]` - массив товаров в корзине. Хранит товары которые клиент добавил для покупки.

Методы класса
`getItems(): IProduct[]` возвращает массив товаров в корзине
`addItem(item: IProduct): void` добавляет переданный товар в корзину
`removeItem(item: IProduct): void` удаляет переданный товар из корзины
`clear(): void` очищает корзину от всех товаров
### BuyerData
Конструктор
`constructor(data: Partial<IBuyer> = {})`
Поля класса
`payment: TPayment | null` выбранный способ оплаты
`email: string` электронная почта покупателя
`phone: string` телефон покупателя
`address: string` адрес покупателя

Методы класса
`saveBuyerData(data: Partial<IBuyer>): void` охраняет переданные данные покупателя
`getBuyer(): IBuyer` возвращает все данные покупателя из модели
`clear(): void` очищает данные покупателя
`validate(): Record<string, string>` - проверяет значение каждого поля, dозвращает объект ошибок

### ApiService

Класс `ApiService` отвечает за получение данных товаров с сервера и за отправку данных заказа на сервер

Конструктор
`constructor(api: IApi)`

Поля класса
`api: IApi` хранит объект для выполнения HTTP запросов

Методы класса
`getProducts(): Promise<IApiProductsResponse>` выполняет GET запрос на ендпоинт `/product/` и возвращает промис объекта с массивом товаров

`submitOrder(data: IOrderData): Promise<IOrderResponse>` принимает объект с данными заказа и выполняет POST запрос на ендпоинт `/order/` возвращает промис объекта


`IApiProductsResponse` объект полученный от сервера при запросе товаров
`IOrderData` объект с данными заказа для отправки на сервер
`IOrderResponse` объект подтверждения полученный от ервера после отправки заказа

### View классы 

1. Card.ts - Базовый класс для всех карточек товара
   - Отображает название, изображение, категорию, цену

2. CardCatalog.ts - Карточка товара в каталоге главной страницы
   - Наследует Card
   - Генерирует событие `card:select` при клике

3. CardPreview.ts*- Карточка товара в модальном окне просмотра
   - Наследует Card
   - Показывает полную информацию о товаре
   - Кнопка "В корзину" / "Удалить из корзины"
   - Генерирует событие `card:buy` при нажатии кнопки

4. Gallery.ts - Контейнер для отображения каталога товаров
   - Базовый класс Component
   - Очищает контейнер при рендере

5. Header.ts - Заголовок страницы с иконкой корзины
   - Отображает счётчик товаров в корзине
   - Генерирует событие `basket:open` при клике на корзину

6. Modal.ts - Модальное окно
   - Управляет видимостью окна (класс `modal_active`)
   - Закрывается по клику на крестик
   - Закрывается по клику вне окна
   - Содержит методы open() и close()

7. Basket.ts - Отображение содержимого корзины
   - Показывает список товаров с номерами
   - Кнопка удаления для каждого товара
   - Общая стоимость
   - Кнопка "Оформить"
   - Сообщение "Корзина пуста"

8. Form.ts - Базовый класс для всех форм
   - Управление видимостью кнопки отправки
   - Отображение ошибок валидации
   - Метод setErrors() для обновления ошибок

9. OrderForm.t - Форма адреса доставки и способа оплаты
   - Наследует Form
   - Выбранная кнопка получает класс `button_alt-active`
   - Генерирует события `order:address`, `order:payment`, `order:submit`

10. ContactsForm.ts - Форма контактных данных
    - Наследует Form
    - Два текстовых поля
    - Генерирует события `contacts:email`, `contacts:phone`, `contacts:submit`

11. SuccessView.ts - Экран успешного оформления заказа
    - Показывает сообщение с суммой списания
    - Генерирует событие `success:close`
